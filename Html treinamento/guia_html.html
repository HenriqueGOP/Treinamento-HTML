01

Estrutura base do documento

Todo arquivo HTML começa com a mesma estrutura base. A primeira linha é sempre o <!DOCTYPE html>, que avisa ao navegador que o arquivo usa HTML5.

Em seguida vem a tag <html>, que é a raiz do documento — ela envolve absolutamente tudo. Ela recebe o atributo lang="pt-br" para indicar o idioma da página.

Dentro do <html> existem dois grandes blocos:

  • <head> — área invisível com configurações da página
  • <body> — tudo que o usuário vê na tela
<!DOCTYPE html> <html lang="pt-br"> <head> <!-- configurações da página --> </head> <body> <!-- conteúdo visível --> </body> </html>

Uma página em branco. A estrutura base por si só não gera elementos visuais na tela, ela apenas prepara o terreno.

02

O <head> e suas tags

Dentro do <head> ficam os metadados — informações sobre a página que não aparecem na tela, mas são essenciais para o funcionamento correto.

<meta charset="UTF-8">

Define a codificação de texto. Sem ela, acentos e caracteres especiais do português podem aparecer quebrados.

<meta name="viewport">

Garante que a página se adapte corretamente ao tamanho de celulares e tablets (responsividade).

<title>

Define o nome que aparece na aba do navegador. É importante tanto para o usuário identificar a página quanto para o Google.

<head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Nome da Página</title> <link rel="stylesheet" href="style.css"> </head>
Nome da Página

O <title> muda o nome que aparece na aba superior do navegador.

03

Organização semântica do <body>

O <body> é dividido em seções com significado próprio, chamadas de tags semânticas. Elas ajudam tanto o navegador quanto leitores de tela a entenderem a estrutura da página.

Tags semânticas principais

  • <header> — cabeçalho visual da página (título principal, menu de navegação)
  • <main> — conteúdo principal, deve existir apenas um por página
  • <footer> — rodapé (copyright, contato, links úteis)
  • <section> — divide o conteúdo em blocos temáticos dentro do <main>
<body> <header> <h1>Título do Site</h1> </header> <main> <section> <!-- bloco de conteúdo --> </section> </main> <footer> <!-- rodapé --> </footer> </body>
<header> Cabeçalho (Menu, Logo)
<main> Conteúdo Principal
<section> Seção de assunto específico
<footer> Rodapé (Copyright, Links)
04

Títulos e parágrafos

Os títulos vão de <h1> até <h6> em ordem decrescente de importância. Use-os como a hierarquia de um livro: capítulos, seções, subseções.

  • <h1> — título principal da página, deve existir apenas um
  • <h2> — subtítulos de seções
  • <h3> — sub-subtítulos
  • <h4> a <h6> — níveis ainda mais profundos

O <p> cria um parágrafo de texto. O navegador adiciona automaticamente um espaço acima e abaixo de cada parágrafo.

<h1>Título Principal</h1> <h2>Subtítulo de Seção</h2> <h3>Sub-subtítulo</h3> <p>Um parágrafo de texto normal.</p>

05

Formatação de texto

Existem várias tags para formatar texto com diferentes significados:

Negrito e itálico

  • <strong> — negrito com significado semântico (texto importante). Use sempre que possível.
  • <b> — negrito apenas visual, sem indicar importância
  • <em> — itálico com ênfase semântica

Outras tags de formatação

  • <br> — quebra de linha (não tem tag de fechamento)
  • <abbr title="..."> — define uma abreviação; o texto do atributo title aparece como dica ao passar o mouse

Comentários

Comentários são escritos com <!-- texto --> e não aparecem na página — servem apenas como anotações para o programador.

<strong>Texto importante</strong> <em>Texto com ênfase</em> Texto com<br>quebra de linha <abbr title="HyperText Markup Language">HTML</abbr> <!-- Isso é um comentário -->

06

Links e imagens

Links — <a>

O <a href="..."> cria um link clicável. Atributos principais:

  • href — endereço de destino do link
  • target="_blank" — abre o link em uma nova aba

Imagens — <img>

A tag <img> exibe uma imagem e não tem tag de fechamento. Atributos principais:

  • src — caminho ou URL da imagem
  • alt — texto alternativo (exibido se a imagem não carregar; lido por leitores de tela)
  • width — largura em pixels

Legenda — <figure> e <figcaption>

O <figure> envolve a imagem e o <figcaption> adiciona uma legenda abaixo. O <figcaption> precisa estar dentro de um <figure>.

<a href="https://google.com" target="_blank">Abrir no Google</a> <figure> <img src="foto.png" alt="Descrição da imagem" width="300"> <figcaption>Legenda da imagem</figcaption> </figure>

Abrir no Google

07

Formulários

O <form> agrupa todos os campos de um formulário. Dentro dele ficam os inputs — campos onde o usuário interage.

Tipos de <input>

O comportamento do <input> muda completamente com o atributo type:

type="text"
Campo de texto livre
type="password"
Esconde o que é digitado
type="email"
Valida formato de e-mail
type="number"
Apenas números (min/max)
type="url"
Valida formato de URL
type="radio"
Escolha única do grupo
type="checkbox"
Marcar múltiplas opções
type="file"
Upload de arquivo
type="date"
Calendário de data
type="time"
Seletor de horário
type="month"
Seletor de mês/ano
type="week"
Seletor de semana
type="range"
Barra deslizante (slider)
type="color"
Seletor de cor
type="submit"
Botão de envio

Atributos importantes do <input>

  • placeholder — texto cinza de dica quando o campo está vazio
  • id — identificador único do elemento
  • name — nome do campo ao enviar o formulário
  • value — valor do campo

Outros elementos de formulário

  • <label for="id"> — texto clicável que ativa o input correspondente
  • <textarea> — campo de texto longo (atributos rows e cols)
  • <select> + <option> — menu suspenso
  • <fieldset> — agrupa campos com uma borda ao redor
  • <legend> — título do fieldset, aparece na borda superior
<form> <fieldset> <legend>Dados pessoais</legend> <label for="nome">Nome:</label> <input type="text" id="nome" name="nome" placeholder="Seu nome..."> <select name="opcao"> <option value="a">Opção A</option> </select> <input type="submit" value="Enviar"> </fieldset> </form>

08

Tabelas

As tabelas são criadas com a tag <table> e divididas em três partes semânticas:

  • <thead> — cabeçalho da tabela
  • <tbody> — corpo com os dados
  • <tfoot> — rodapé da tabela

Células

  • <tr> — cria uma linha
  • <th> — célula de cabeçalho (negrito e centralizado automaticamente)
  • <td> — célula normal de dados
  • colspan="N" — faz uma célula ocupar o espaço de N colunas
<table> <thead> <tr> <th>Coluna 1</th> <th>Coluna 2</th> </tr> </thead> <tbody> <tr> <td>Dado A</td> <td>Dado B</td> </tr> </tbody> <tfoot> <tr> <td colspan="2">Rodapé</td> </tr> </tfoot> </table>

09

Listas

O HTML tem dois tipos de lista. Em ambos os casos, cada item é marcado com <li>.

  • <ol> — lista ordenada, os itens são numerados automaticamente
  • <ul> — lista não ordenada, os itens aparecem com bolinhas
<ol> <li>Primeiro item</li> <li>Segundo item</li> </ol> <ul> <li>Item sem ordem</li> <li>Outro item</li> </ul>

  2. Segundo item

10

Conteúdo expansível e outros elementos

<details> e <summary>

O <details> cria um bloco que pode ser aberto e fechado com um clique pelo usuário. O <summary> define o texto clicável que aparece como título.

<span>

O <span> é um container inline (não quebra linha). É usado para envolver um trecho de texto para aplicar estilos com CSS ou capturar valores com JavaScript.

Entidades HTML

São códigos especiais para representar caracteres que o HTML reserva ou que não podem ser digitados diretamente:

  • &copy; → © (símbolo de copyright)
  • &lt; → < (menor que, pois o sinal bruto seria interpretado como tag)
  • &gt; → > (maior que)
  • &amp; → & (e comercial)
<details> <summary>Clique para ver mais</summary> <p>Conteúdo escondido que aparece ao clicar.</p> </details> <p>Dificuldade: <span id="valor">5</span>/10</p> <footer> <p>&copy; 2026 Meu Site</p> </footer>
💡 Dica: O <span> sozinho não faz nada visualmente — ele é uma "caixa invisível" que só tem utilidade quando combinado com CSS ou JavaScript.
